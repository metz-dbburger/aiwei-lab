# aiwei-lab · AI味实验室

你一定见过这些句子：

> 这不仅仅是 X，而是 Y。
> 我要说一个震惊所有人的事情，然而没有人发现。
> 真相一旦揭开，会让所有人大跌眼镜。

为什么 AI 张嘴就是这一套？这个仓库让你**自己动手体验**一种解释：
**一部分AI味，可能是已有表达被偏好训练进一步放大的结果**——人类先发明了
这些廉价修辞，偏好标注中它们更容易被选中（典型性偏差），对齐训练可能进一步
放大它们，使用合成数据的训练管线可能把它们继续传递下去。

单个 Python 文件，前三个实验**零依赖、不用任何 API key**。

## 实验一：模板计数器

```bash
python3 aiweilab.py demo            # 内置语料对比
python3 aiweilab.py scan 文章.txt    # 扫你自己的文本
```

统计 8 类特征的每千字密度：对比否定（不是A而是B）、震惊体、划重点腔、
AI 连接词、破折号、万能升华、delve 类英文词、排比开头。

演示口径（重要）：内置语料是**刻意构造的样本**——素人文本故意不加修辞，AI 文本刻意堆满模板，
后者合计 57.85 次/千字、前者接近 0。这验证的是计数器本身能量出差距，
**不代表一般 AI 文本与人类文本的普遍差距**。有意义的用法是扫你自己的真实文本。

> ⚠️ 它是频率对比工具，**不是 AI 测谎仪**。低分不证明是人写的，
> 高分不证明是 AI 写的。请不要拿它指控任何作者——AI 检测器误伤
> 真人作者的事故已经够多了。

## 实验二：你也是奖励模型

```bash
python3 aiweilab.py quiz
```

8 组**演示句对**：每组保留同一个核心事实，其中一句叠加常见修辞框架
（"很少有人知道，""不是别的，正是"）。单一指标：哪句你更愿意继续读下去。

这是一次借用 RLHF 偏好标注形式的**体验**（两个候选，选一个），不是论文实验的复现，
单人结果不构成统计证据。相关研究：2025 年有团队用真人被试重演标注流程
（被试主要来自全球南方、约九成来自非洲），
发现被试系统性偏爱含焦点词的文本变体（arXiv:2508.01930）。

## 实验三：口述采样（零代码）

```bash
python3 aiweilab.py vs   # 打印操作说明
```

在任何聊天 AI 上做两轮对比：直接要 1 句开场白（重复 5 次，**每次新开会话**），
vs 一次性要"5 个候选+各自概率"。对比两轮各自的 5 句：相似程度是否不同，
模板句出现的频率是否不同。
出处：Verbalized Sampling（arXiv:2510.01171）；1.6–2.1 倍是论文在创意写作任务上的实测口径，
单人演示能做的只是观察两轮结果是否不同，识别不了原因；
论文作者的解释是模型仍保留了一部分默认回答之外的分布，这一点无法用单人演示验证。

## 实验四：对齐前后对照（可选，需装依赖）

```bash
pip install transformers torch
python3 aiweilab.py probe --dry   # 先看实验设计
python3 aiweilab.py probe         # CPU 可跑，首次下载两个 0.5B 模型
```

指标按**题内计算后跨题平均**（避免题目间天然差异干扰）；`--seed` 固定随机种子（默认 42）；
原始输出连同模型名与种子保存到 `probe_outputs.json` 供复核。这是**探索性演示**。

同一组问题，各采样 12 次，对比 Qwen2.5-0.5B 基座版与对齐版：

| 指标 | 预期 |
|---|---|
| 模板命中率 | 对齐后 ↑ |
| distinct-2 多样性 | 对齐后 ↓ |
| 开头四字去重率 | 对齐后 ↓ |
| "……不是X，"之后接"而是"的概率 | 对齐后 ↑ |

**解读边界：** 基座版与对齐版之间同时混合了 SFT、偏好优化、聊天模板等差异，
本实验不能单独归因于其中任何一项。要把账单精确
算到偏好优化头上，用 SFT-only 与 DPO 检查点对照（进阶：
`HuggingFaceH4/mistral-7b-sft-beta` vs `HuggingFaceH4/zephyr-7b-beta`）。
这里用 0.5B 小模型只是为了人人能跑；它的结果不能直接外推到大模型，本仓库没有做过大模型实测。

## 背后的研究

- Kirk et al., *Understanding the Effects of RLHF on LLM Generalisation and Diversity*, ICLR 2024（arXiv:2310.06452）——RLHF 显著降低输出多样性
- Juzek & Ward, *Why Does ChatGPT "Delve" So Much?*, COLING 2025（arXiv:2412.11385）——排除若干直接解释后，结果与人类反馈的参与一致（证据混杂，原文口径克制）
- Juzek et al., *Word Overuse and Alignment in LLMs*（arXiv:2508.01930）——真人重演标注流程，为“偏好信号可传导词汇偏好”补上实验支撑（作者声明仍有其他因素待研究）
- *Verbalized Sampling*（arXiv:2510.01171）——提出典型性偏差作为坍缩的数据层解释，并显示提示方式可提高观察到的多样性
